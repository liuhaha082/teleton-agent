Plugin Development Guide

Extend the agent with custom tools, event hooks, and background tasks. Plugins are ESM JavaScript modules placed in ~/.teleton/plugins/.

Overview

Key facts about plugins:

  • Plugins receive a frozen SDK object: they cannot modify or extend it
  • Each plugin gets an isolated SQLite database (if migrate is exported)
  • Plugins see a sanitized config with no API keys or secrets
  • The official SDK package is @teleton-agent/sdk on npm
  • Plugins are loaded at startup, no rebuild needed

Plugin Structure

Plugins can be a single file or a directory:

Single file
~/.teleton/plugins/
  my-plugin.js          # Self-contained plugin
Directory (recommended for plugins with dependencies)
~/.teleton/plugins/
  my-plugin/
    index.js            # Entry point (required)
    package.json        # npm dependencies (optional)
    package-lock.json   # Lockfile (required if package.json exists)
    node_modules/       # Auto-installed by the platform

When a plugin has a package.json and package-lock.json, the platform automatically runs npm ci --ignore-scripts to install dependencies before loading.

Minimal Plugin (Hello World)

~/.teleton/plugins/hello.js
export const tools = [
  {
    name: "hello_greet",
    description: "Greet a user by name",
    parameters: {
      type: "object",
      properties: {
        name: { type: "string", description: "The person's name" },
      },
      required: ["name"],
    },
    async execute(params) {
      return {
        success: true,
        data: { message: `Hello, ${params.name}!` },
      };
    },
  },
];

Restart the agent (or enable hot-reload) and the hello_greet tool will be available to the LLM.

Manifest

manifest example
export const manifest = {
  name: "my-plugin",           // Required: lowercase, alphanumeric + hyphens
  version: "1.0.0",            // Required: semver
  author: "Your Name",
  description: "What it does", // Max 256 chars
  dependencies: ["deals"],     // Required built-in modules
  sdkVersion: ">=1.0.0",
  defaultConfig: {
    max_results: 10,
    cooldown_ms: 5000,
  },
  secrets: {
    api_key: {
      required: true,
      description: "External API key for the service",
    },
  },
};

Plugin config is resolved by merging manifest.defaultConfig with the user's config under plugins.<plugin_name> in config.yaml (hyphens replaced by underscores).

Full Plugin Example

~/.teleton/plugins/weather.js
export const manifest = {
  name: "weather",
  version: "1.0.0",
  sdkVersion: "^1.0.0",
};

// Creates isolated database at ~/.teleton/plugins/data/weather.db
export function migrate(db) {
  db.exec(`CREATE TABLE IF NOT EXISTS weather_cache (
    city TEXT PRIMARY KEY, data TEXT, cached_at INTEGER
  )`);
}

// Required: tools as a function receiving the Plugin SDK
export const tools = (sdk) => [
  {
    name: "weather_get",
    description: "Get current weather for a city",
    parameters: {
      type: "object",
      properties: { city: { type: "string", description: "City name" } },
      required: ["city"],
    },
    execute: async (params) => {
      sdk.log.info(`Fetching weather for ${params.city}`);
      const res = await fetch(`https://wttr.in/${params.city}?format=j1`);
      if (!res.ok) return { success: false, error: "City not found" };
      const data = await res.json();
      return { success: true, data: { temp: data.current_condition[0].temp_C } };
    },
  },
];

Hot-Reload During Development

Enable hot-reload in config.yaml so the platform watches ~/.teleton/plugins/ for file changes and automatically reloads modified plugins without restarting:

config.yaml
dev:
  hot_reload: true

Managing Plugin Secrets

Plugin API keys and tokens should NOT be stored in config.yaml. Use the /plugin admin command instead:

Telegram
/plugin set weather API_KEY your-api-key-here
/plugin keys weather        # List configured secret keys

Secrets are stored in ~/.teleton/plugins/data/<plugin>.secrets.json with 0600 permissions. Access them in your plugin via sdk.secrets.get("API_KEY").

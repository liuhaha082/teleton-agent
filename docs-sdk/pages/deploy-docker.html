Deployment Guide

Every method of deploying Teleton Agent: npm global install, Docker, Docker Compose, from source, and systemd for VPS.

Prerequisites

  • Node.js 20+ (required by engines field in package.json)
  • npm (ships with Node.js)
  • Telegram API credentials: api_id and api_hash from my.telegram.org/apps
  • LLM API key: from your chosen provider
  • Build tools (only for source/Docker builds): python3, make, g++ for native modules

Method 1: npm Global Install

The simplest deployment path. Suitable for personal use and quick testing.

npm global install
# Install globally
npm install -g teleton

# Run the interactive setup wizard
teleton setup

# Start the agent
teleton start

On first launch, Telegram will send a login code to your phone. Enter it when prompted. After successful authentication, a session file is saved at ~/.teleton/ and subsequent launches will not require re-authentication.

Method 2: Docker

The official Docker image is available on GitHub Container Registry.

Pull and run
docker run -d \
  --name teleton \
  --restart unless-stopped \
  -v teleton-data:/data \
  -p 7777:7777 \
  ghcr.io/tonresistor/teleton-agent

Image Details

  • Data volume: Mounted at /data (set via TELETON_HOME=/data in the image)
  • Entrypoint: node dist/cli/index.js
  • Default command: start
  • Exposed port: 7777 (WebUI, when enabled)
  • Runs as: non-root node user

Interactive Setup with Docker

Docker interactive setup then run
# Run setup interactively
docker run -it --rm \
  -v teleton-data:/data \
  ghcr.io/tonresistor/teleton-agent setup

# Then start the agent
docker run -d \
  --name teleton \
  --restart unless-stopped \
  -v teleton-data:/data \
  -p 7777:7777 \
  ghcr.io/tonresistor/teleton-agent

Passing Configuration via Environment

Docker with env vars
docker run -d \
  --name teleton \
  --restart unless-stopped \
  -e TELETON_API_KEY="sk-ant-..." \
  -e TELETON_TG_API_ID="12345678" \
  -e TELETON_TG_API_HASH="0123456789abcdef" \
  -e TELETON_TG_PHONE="+1234567890" \
  -e TELETON_WEBUI_ENABLED="true" \
  -v teleton-data:/data \
  -p 7777:7777 \
  ghcr.io/tonresistor/teleton-agent

Method 3: Docker Compose

docker-compose.yml
version: "3.8"

services:
  teleton:
    image: ghcr.io/tonresistor/teleton-agent:latest
    container_name: teleton
    restart: unless-stopped
    ports:
      - "7777:7777"  # WebUI (remove if not using)
    volumes:
      - teleton-data:/data
    environment:
      - TELETON_WEBUI_ENABLED=true
      - TELETON_WEBUI_HOST=0.0.0.0
      # - TELETON_API_KEY=sk-ant-...
      # - TELETON_TG_API_ID=12345678
      # - TELETON_TG_API_HASH=0123456789abcdef
      # - TELETON_TG_PHONE=+1234567890
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:7777/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 30s

volumes:
  teleton-data:
    driver: local
Docker Compose commands
# Initial setup (interactive)
docker compose run --rm teleton setup

# Start in background
docker compose up -d

# View logs
docker compose logs -f teleton

# Update to latest image
docker compose pull && docker compose up -d

Method 4: From Source

Build from source
git clone https://github.com/TONresistor/teleton-agent.git
cd teleton-agent

npm install
cd web && npm install && cd ..
npm run build

node dist/cli/index.js setup
node dist/cli/index.js start

Development Mode

Dev mode with auto-reload
# Backend with tsx watch
npm run dev

# Frontend dev server (separate terminal)
npm run dev:web

systemd Service (VPS)

/etc/systemd/system/teleton.service
[Unit]
Description=Teleton AI Agent for Telegram
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=teleton
Group=teleton
WorkingDirectory=/home/teleton

ExecStart=/usr/bin/teleton start
# Or: ExecStart=/usr/bin/node /home/teleton/teleton-agent/dist/cli/index.js start

Restart=on-failure
RestartSec=10

Environment=NODE_ENV=production
Environment=TELETON_HOME=/home/teleton/.teleton

# Security hardening
NoNewPrivileges=true
ProtectSystem=strict
ProtectHome=read-only
ReadWritePaths=/home/teleton/.teleton
PrivateTmp=true

StandardOutput=journal
StandardError=journal
SyslogIdentifier=teleton

[Install]
WantedBy=multi-user.target
Enable and start
sudo useradd -r -m -s /bin/bash teleton
sudo npm install -g teleton
sudo systemctl daemon-reload
sudo systemctl enable teleton
sudo systemctl start teleton
sudo journalctl -u teleton -f

Backup Strategy

All persistent data is in ~/.teleton/ (or TELETON_HOME). Regular backups of this directory are sufficient for full recovery.

PathContentsSensitivity
config.yamlAll configurationContains API keys
wallet.jsonTON wallet mnemonic + keysHighly sensitive (0600 perms)
teleton_session/Telegram sessionGrants account access
memory.dbConversation memory + sessions (SQLite)Contains chat history
plugins/Installed pluginsReproducible
workspace/Agent workspace filesUser content
Backup commands
# Simple backup
tar -czf teleton-backup-$(date +%Y%m%d).tar.gz ~/.teleton/

# Exclude transient files
tar -czf teleton-backup-$(date +%Y%m%d).tar.gz \
  --exclude='*.db-wal' \
  --exclude='*.db-shm' \
  ~/.teleton/

Configuration Reference

Teleton Agent is configured through a single YAML file at ~/.teleton/config.yaml. Run teleton setup to generate it interactively.

agent

LLM provider and agentic loop configuration.

KeyTypeDefaultDescription
agent.providerenum"anthropic"LLM provider. One of: anthropic, claude-code, openai, google, xai, groq, openrouter, moonshot, mistral, cerebras, zai, minimax, huggingface, cocoon, local.
agent.api_keystring""API key for the chosen provider. Can be overridden with TELETON_API_KEY env var.
agent.modelstring"claude-haiku-4-5-20251001"Primary model ID.
agent.utility_modelstringauto-detectedCheap/fast model for summarization and compaction.
agent.base_urlstringoptionalBase URL for local LLM server (e.g., http://localhost:11434/v1).
agent.max_tokensnumber4096Maximum tokens in each LLM response.
agent.temperaturenumber0.7Sampling temperature (0.0 = deterministic, 1.0 = creative).
agent.max_agentic_iterationsnumber5Maximum agentic loop iterations per message.

agent.session_reset_policy

KeyTypeDefaultDescription
daily_reset_enabledbooleantrueEnable automatic daily session reset.
daily_reset_hournumber4Hour of day (0-23) to reset all sessions.
idle_expiry_enabledbooleantrueEnable session reset after inactivity.
idle_expiry_minutesnumber1440Minutes of inactivity before session resets (default: 24h).

telegram

Telegram client and messaging behavior.

KeyTypeDefaultDescription
telegram.api_idnumber(required)Telegram API ID from my.telegram.org/apps.
telegram.api_hashstring(required)Telegram API hash.
telegram.phonestring(required)Phone number in international format (e.g., "+1234567890").
telegram.dm_policyenum"allowlist"Who can interact via DMs. See policies below.
telegram.allow_fromnumber[][]Telegram user IDs allowed to DM (used with "allowlist").
telegram.group_policyenum"open"Who can interact in groups.
telegram.require_mentionbooleantrueIn groups, only respond when mentioned by name/username.
telegram.admin_idsnumber[][]Telegram user IDs with admin privileges.
telegram.debounce_msnumber1500Debounce delay for group messages. Set to 0 to disable.
telegram.bot_tokenstringoptionalBot token for inline buttons (deals system).
telegram.owner_namestringoptionalOwner's first name (used in personality prompts).
telegram.owner_usernamestringoptionalOwner's Telegram username without @.

DM Policies

ValueBehavior
"allowlist"Only users listed in allow_from can interact. Default.
"open"Anyone can DM the agent.
"admin-only"Only users in admin_ids can interact via DM.
"disabled"DMs are completely ignored.

Group Policies

ValueBehavior
"open"Agent responds in any group it is a member of. Default.
"allowlist"Only responds in groups listed in group_allow_from.
"admin-only"Only responds when triggered by users in admin_ids.
"disabled"Group messages are completely ignored.

webui

Optional web dashboard for monitoring and management.

KeyTypeDefaultDescription
webui.enabledbooleanfalseEnable the WebUI server.
webui.portnumber7777HTTP server port.
webui.hoststring"127.0.0.1"Bind address. Set to "0.0.0.0" to expose externally (use with a reverse proxy).
webui.auth_tokenstringauto-generatedBearer token for API authentication.

deals

Configuration for the peer-to-peer deals/escrow system.

KeyTypeDefaultDescription
deals.enabledbooleantrueEnable the deals module.
deals.expiry_secondsnumber120Time before an unaccepted deal expires.
deals.buy_max_floor_percentnumber95Maximum price as a percentage of floor for buy deals.
deals.sell_min_floor_percentnumber105Minimum price as a percentage of floor for sell deals.

heartbeat

Periodic timer that triggers the agent to check for pending tasks.

KeyTypeDefaultDescription
heartbeat.enabledbooleantrueEnable the periodic heartbeat timer.
heartbeat.interval_msnumber3600000Heartbeat interval in milliseconds (minimum 60000).
heartbeat.promptstringsee docsPrompt sent to the agent on each heartbeat tick.
heartbeat.self_configurablebooleanfalseAllow the agent to modify heartbeat config at runtime.

mcp

Model Context Protocol (MCP) server configuration. Supports stdio, SSE, and Streamable HTTP transports.

mcp example
mcp:
  servers:
    filesystem:
      command: "npx @modelcontextprotocol/server-filesystem /tmp"
      scope: "admin-only"
    remote-api:
      url: "https://mcp.example.com/sse"
      enabled: true

capabilities

Controls optional agent capabilities that require explicit opt-in.

YOLO mode (exec)
capabilities:
  exec:
    mode: "yolo"        # off | yolo
    scope: "admin-only" # admin-only | allowlist | all
    limits:
      timeout: 120
      max_output: 50000
    audit:
      log_commands: true

Environment Variable Overrides

Environment variables override values set in config.yaml.

VariableConfig PathDescription
TELETON_HOME-Root data directory (default: ~/.teleton)
TELETON_API_KEYagent.api_keyOverride the LLM provider API key
TELETON_TG_API_IDtelegram.api_idOverride the Telegram API ID
TELETON_TG_API_HASHtelegram.api_hashOverride the Telegram API hash
TELETON_TG_PHONEtelegram.phoneOverride the phone number
TELETON_WEBUI_ENABLEDwebui.enabledEnable WebUI ("true" or "false")
TELETON_WEBUI_PORTwebui.portWebUI server port
TELETON_API_ENABLEDapi.enabledEnable Management API
TELETON_TAVILY_API_KEYtavily_api_keyTavily API key for web search
TELETON_TONAPI_KEYtonapi_keyTonAPI key for blockchain queries

Precedence Order

  1. CLI flags (--webui, -c): highest priority
  2. Environment variables (TELETON_* overrides)
  3. Config file (config.yaml)
  4. Schema defaults: lowest priority

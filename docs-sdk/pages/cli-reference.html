CLI Reference

The teleton CLI manages the agent lifecycle, configuration, and MCP servers.

Usage
teleton <command> [options]
teleton --help
teleton --version

teleton setup

Interactive wizard to configure Teleton for the first time. Run this before teleton start. Prompts for Telegram API credentials, phone number, and LLM provider key.

Example
# Interactive setup
teleton setup

# Headless setup via Web UI
teleton setup --ui --ui-port 7777

# Fully non-interactive (CI/CD)
teleton setup --non-interactive \
  --api-id 12345 \
  --api-hash abc123 \
  --phone +1234567890 \
  --api-key sk-ant-...
FlagDefaultDescription
--workspace <dir>~/.teletonWorkspace directory
--non-interactivefalseSkip all prompts (requires all values as flags)
--uifalseLaunch web-based setup wizard in browser
--ui-port <port>7777Port for the setup WebUI
--api-id <id>-Telegram API ID (from my.telegram.org)
--api-hash <hash>-Telegram API Hash
--phone <number>-Phone number with country code (e.g. +1234567890)
--api-key <key>-LLM provider API key (Anthropic, OpenAI, etc.)
--base-url <url>-Base URL for local LLM server (Ollama, LM Studio)
--user-id <id>-Telegram User ID to set as admin
--tavily-api-key <key>-Tavily API key for web search (free at tavily.com)

teleton start

Start the Teleton agent. Requires a valid config file (run teleton setup first). With --api and no config, starts in bootstrap mode for remote setup.

Example
# Normal start
teleton start

# With custom config path
teleton start -c /etc/teleton/config.yaml

# Enable WebUI on port 8080
teleton start --webui --webui-port 8080

# Enable Management API and output credentials as JSON
teleton start --api --json-credentials

# Bootstrap mode (no config needed, useful for first-time remote setup)
teleton start --api --api-port 7778
FlagDefaultDescription
-c, --config <path>~/.teleton/config.yamlConfig file path
--webuifalseEnable WebUI server (overrides config)
--webui-port <port>7777WebUI server port
--apifalseEnable Management API server (overrides config)
--api-port <port>7778Management API port
--json-credentialsfalsePrint API credentials as JSON to stdout on start

teleton doctor

Run system health checks and report the status of all components: Node.js version, config file, Telegram credentials, API key, session, wallet, SOUL.md, memory database, AI model, and admin users.

Example
teleton doctor

# Output:
#   ✓ Node.js: v22.0.0
#   ✓ Config file: Valid
#   ✓ Telegram credentials: Phone: +1234567890
#   ✓ Anthropic API key: sk-a****...1234
#   ✓ Telegram session: Active (today)
#   ✓ TON wallet: EQAbc...xyz
#   ✓ SOUL.md: 3.2 KB
#   ✓ Memory database: 0.42 MB
#   ✓ AI Model: anthropic/claude-haiku-4-5-20251001
#   ✓ Admin users: 1 user: 123456789
FlagDefaultDescription
No flags. Uses default workspace at ~/.teleton.

teleton mcp

Manage MCP (Model Context Protocol) servers. MCP servers extend the agent with additional tools. Restart the agent after changes.

teleton mcp add <package> [args...]

Add an MCP server. The server name is auto-derived from the package name. Use --url for SSE/HTTP servers.

Example
# Add filesystem server with root at /tmp
teleton mcp add @modelcontextprotocol/server-filesystem /tmp

# Add with custom name and scope
teleton mcp add @modelcontextprotocol/server-brave-search \
  --name brave \
  --scope dm-only \
  --env BRAVE_API_KEY=your-key

# Add SSE/HTTP server by URL
teleton mcp add https://mcp.example.com/sse --url
FlagDefaultDescription
-n, --name <name>auto-derivedServer name (derived from package if omitted)
-s, --scope <scope>alwaysTool scope: always | open | dm-only | group-only | admin-only | allowlist | disabled
-e, --env <KEY=VALUE>-Environment variables (repeatable: -e A=1 -e B=2)
--urlfalseTreat <package> as an SSE/HTTP URL instead of npx package
-c, --config <path>default configConfig file path

teleton mcp remove <name>

Remove an MCP server by name.

Example
teleton mcp remove filesystem
FlagDefaultDescription
-c, --config <path>default configConfig file path

teleton mcp list

List all configured MCP servers with their type, scope, and env vars.

Example
teleton mcp list

# Output:
# MCP servers (2):
#   ✓ filesystem (stdio, always)
#     npx
#     env: (none)
#   ✓ brave (stdio, dm-only)
#     npx
#     env: BRAVE_API_KEY
FlagDefaultDescription
-c, --config <path>default configConfig file path

teleton config <action> [key] [value]

Manage individual configuration keys. Only whitelisted keys are configurable. Sensitive values (API keys, tokens) are masked in output.

ActionUsageDescription
listteleton config listShow all configurable keys and their current values
getteleton config get <key>Show the current value of a key
setteleton config set <key> [value]Set a key's value. Prompts interactively if value omitted.
unsetteleton config unset <key>Remove a key's value
Example
# List all settings
teleton config list

# Get a value
teleton config get tavily_api_key

# Set interactively (prompts securely for sensitive values)
teleton config set telegram.bot_token

# Set inline
teleton config set agent.model claude-sonnet-4-6

# Remove a value
teleton config unset tavily_api_key
FlagDefaultDescription
-c, --config <path>default configConfig file path

API Management

teleton api-rotate-key

Generate a new Management API key. The plaintext key is printed to stdout exactly once, store it securely. The hash is saved to config.

Example
teleton api-rotate-key

# Output (example):
# tltn_Abc123...XYZ789

# Store in env or password manager immediately
FlagDefaultDescription
-c, --config <path>default configConfig file path

teleton api-fingerprint

Show the TLS certificate SHA-256 fingerprint for the Management API. Use this to verify the server identity in API clients.

Example
teleton api-fingerprint

# Output (example):
# AA:BB:CC:DD:EE:FF:00:11:22:33:44:55:66:77:88:99:AA:BB:CC:DD:EE:FF:00:11:22:33:44:55:66:77:88:99
FlagDefaultDescription
No flags.
